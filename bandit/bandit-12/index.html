Bandit 12 / Overthewire

October 29, 2023

  • bandit
  • ssh
  • sshpass
  • linux
  • bash

Información

The password for the next level is stored in the file data.txt, which is a hexdump of a file that has been repeatedly compressed. For this level it may be useful to create a directory under /tmp in which you can work using mkdir. For example: mkdir /tmp/myname123. Then copy the datafile using cp, and rename it using mv (read the manpages!)

Resolución

Antes de empezar el nivel, recomendaría lavarte la cara y que te de un poco el aire, ya que es un ejercicio largo y con muchos comandos nuevos. ¡Suerte!

Creando entorno de trabajo

Como en este ejercicio necesitamos modificar y crear nuevos archivos tendremos que crearnos un directorio en /tmp:

bandit12@bandit:~$ mkdir /tmp/ex12
bandit12@bandit:~$ cd /tmp/ex12

bandit12@bandit:/tmp/ex12$ cp ~/data.txt ./.

bandit12@bandit:/tmp/ex12$ ls
data.txt

Comprimiendo a hexadecimal / Primer paso

Cuando hacemos un cat data.txt podemos identificar que es hexadecimal, una vez con esta información desomprimimos el fichero .txt que en verdad es un binario de hexdump, para hacerlo usamos el comando: xxd -r y redirigimos el output al nuevo fichero data.

Con el comando file vemos que tiene encriptado otro archivo llamado data2.bin, como un winrar que tiene un archivo comprimido…

bandit12@bandit:/tmp/ex12$ cat data.txt | xxd -r > data

bandit12@bandit:/tmp/ex12$ file data
data: gzip compressed data, was "data2.bin", last modified: Sun Apr 23 18:04:23 2023, max compression, from Unix, original size modulo 2^32 581

Hemos sacado el archivo fuera de la capa de hexadecimal, ahora hacemos un file para ver que tipo de archivo es y vemos que esta comprimido con gzip, entonces cambiamos la extensión a .gz (es decir, el archivo comprimido con gzip).

Después de hacer un file al nuevo data, nos damos cuenta que esta comprimido con bzip2.

bandit12@bandit:/tmp/ex12$ gzip -d data.gz 
bandit12@bandit:/tmp/ex12$ file data
data: bzip2 compressed data, block size = 900k

Cambiando extensión: bz2 / Segundo paso

Ahora tendremos que cambiar la entensión y poner: bzip2 -d data.bz2, y así continuamente…

bandit12@bandit:/tmp/ex12$ mv data data.bz2
bandit12@bandit:/tmp/ex12$ bzip2 -d data.bz2 

bandit12@bandit:/tmp/ex12$ file data
data: gzip compressed data, was "data4.bin", last modified: Sun Apr 23 18:04:23 2023, max compression, from Unix, original size modulo 2^32 20480

Cambiando extensión: gz / Tercer paso

Los archivos comprimidos con gzip contienen la extensión: gz.

  • Para descomprimirlos usamos el comando: gzip -d
bandit12@bandit:/tmp/ex12$ mv data data.gz
bandit12@bandit:/tmp/ex12$ gzip -d data.gz 

bandit12@bandit:/tmp/ex12$ file data
data: POSIX tar archive (GNU)

Cambiando extensión: tar / Cuarto paso

Los archivos comprimidos con tar contienen la extensión: tar.

  • Para descomprimirlos usamos el comando: tar xf
bandit12@bandit:/tmp/ex12$ mv data data.tar
bandit12@bandit:/tmp/ex12$ tar xf data.tar
bandit12@bandit:/tmp/ex12$ ls
data5.bin  data.tar  data.txt

bandit12@bandit:/tmp/ex12$ file data5.bin 
data5.bin: POSIX tar archive (GNU)

Cambiando extensión: tar / Quinto paso

Los archivos comprimidos con tar contienen la extensión: tar.

  • Para descomprimirlos usamos el comando: tar xf
bandit12@bandit:/tmp/ex12$ mv data5.bin data5.tar
bandit12@bandit:/tmp/ex12$ tar xf data5.tar
bandit12@bandit:/tmp/ex12$ ls
data5.bin  data.tar  data.txt

bandit12@bandit:/tmp/ex12$ file data5.bin 
data5.bin: POSIX tar archive (GNU)

Cambiando extensión: tar / Sexto paso

Los archivos comprimidos con tar contienen la extensión: tar.

  • Para descomprimirlos usamos el comando: tar xf
bandit12@bandit:/tmp/ex12$ mv data5.bin data5.tar
bandit12@bandit:/tmp/ex12$ tar xf data5.tar 
bandit12@bandit:/tmp/ex12$ ls
data5.tar  data6.bin  data.tar  data.txt

bandit12@bandit:/tmp/ex12$ file data6.bin 
data6.bin: bzip2 compressed data, block size = 900k

Cambiando extensión: bz2 / Séptimo paso

Los archivos comprimidos con bzip2 contienen la extensión: bz2.

  • Para descomprimirlos usamos el comando: bzip2 -d
bandit12@bandit:/tmp/ex12$ mv data6.bin data6.bz2
bandit12@bandit:/tmp/ex12$ bzip2 -d data6.bz2 

bandit12@bandit:/tmp/ex12$ file data6
data6: POSIX tar archive (GNU)

Cambiando extensión: tar / Octavo paso

Los archivos comprimidos con tar contienen la extensión: tar.

  • Para descomprimirlos usamos el comando: tar xf
bandit12@bandit:/tmp/ex12$ mv data6 data6.tar
bandit12@bandit:/tmp/ex12$ tar xf data6.tar 
bandit12@bandit:/tmp/ex12$ ls
data5.tar  data6.tar  data8.bin  data.tar  data.txt

bandit12@bandit:/tmp/ex12$ file data8.bin 
data8.bin: gzip compressed data, was "data9.bin", last modified: Sun Apr 23 18:04:23 2023, max compression, from Unix, original size modulo 2^32 49

Cambiando extensión: gz / Noveno paso

Los archivos comprimidos con gzip contienen la extensión: gz.

  • Para descomprimirlos usamos el comando: gzip -d
bandit12@bandit:/tmp/ex12$ mv data8.bin data8.gz
bandit12@bandit:/tmp/ex12$ gzip -d data8.gz 

bandit12@bandit:/tmp/ex12$ file data8
data8: ASCII text

Leyendo la contraseña / Último paso

bandit12@bandit:/tmp/ex12$ cat data8
The password is wbWdlBxEir4CaE8LaPhauuOo6pwRmrDw
