Información

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

Resolución

Muy parecido al ejercicio anterior.

Encontrar puerto en localhost / Primer paso

Nmap es una herramienta especializada en scaneos de puertos y de sus versiones, esta es una manera sencilla de usaro.

Preparando entorno / Tercer paso

Una vez que tenemos la private key, podemos acceder a bandit 17:

  • !$ -> En este caso es lo mismo que poner /tmp/createkey, nos referimos al último directorio/archivo mencionado
bandit16@bandit:~$ mkdir /tmp/createkey
bandit16@bandit:~$ cd !$
cd /tmp/createkey

Ahora con nano id_rsa (un editor de texto) meteremos la key que nos devolvió el servidor, se almacenara en el archivo id_rsa (lo crea automaticamente):

  • Ctrl + X -> Para guardar los cambios en nano

Damos permisos de ejecución / Cuarto paso

Ahora daremos permisos de ejecución a nuestra llave privada, para hacerlo usamos el comando: chmod:

bandit16@bandit:/tmp/createkey$ chmod 600 id_rsa 

bandit16@bandit:/tmp/createkey$ ls -l id_rsa 
-rw------- 1 bandit16 bandit16 1675 Sep 10 15:00 id_rsa

En este caso usamos chmod 600, es para indicar que el creador del archivo tenga permisos de lectura y escritura (rw-). Para calcular esto hay que saber octal, aquí un pequeño esquema de como calcularlo:

  • r = 4
  • w = 2
  • x = 1

También hay que saber que los permisos van por 3 campos:

  • Algunas veces, si es un directorio, tendrá 10 espacios, el primero con una d para indicarlo

  • Los primeros 3 espacios son para el creador

  • Los siguientes 3 para el grupo del creador

  • Los últimos 3 para otros usuarios, generalmente invitados

Si queremos que un archivo tenga permisos de ejecución y lectura para todos los usuarios, y que el creador sea el único que pueda modificarlo usamos el siguiente comando:

chmod 755 archivo

También podríamos hacerlo de la siguiente manera, pero es menos optimo:

chmod rwxr-xr-x archivo

Nos registramos ya con la llave creada / Último paso

Exactamente como el registro de bandit 14:

bandit16@bandit:/tmp/createkey$ ssh bandit17@localhost -p2220 -i id_rsa 
...
bandit17@bandit:~$ whoami
